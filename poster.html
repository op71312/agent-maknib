ระบบจะให้คะแนน (Score) จากการเปรียบเทียบวิธีการเดินหมากหนีบกับกลยุทธ์เป้าหมาย หาก Score ออกมาสูง แสดงว่าวิธีการเดินหมากหนีบสอดคล้องกับกลยุทธ์เป้าหมายมากขึ้น

1. การตั้งค่าระบบและโหลดโมเดล

ติดตั้งไลบรารี่สำคัญ: pandas, numpy, attacut, sentence_transformers, scikit-learn และโหลดโมเดล Qwen/Qwen3-Embedding-0.6B สำหรับสร้าง embeddings ภาษาไทย

2. สร้าง 6C Framework

กำหนดกรอบการวิเคราะห์ 6 มิติสำหรับเกมหมากหนีบ: ความสามารถในการรุก, ความสามารถในการรับ, การประสานงานระหว่างหมาก, ทรัพยากรที่ยังไม่ใช้, ความได้เปรียบด้านเวลา, ความเหมาะสมกับสถานการณ์

3. การโหลดและประมวลผลข้อมูลกลยุทธ์

โหลดข้อมูลวิธีเดินหมากหนีบที่สอดคล้องกับ 36 กลยุทธ์จากไฟล์ CSV, รวมข้อความ (หมวด + ลักษณะ) เป็น combined_text และตัดคำภาษาไทยด้วย attacut สำหรับการประมวลผลที่แม่นยำ

4. การสร้าง Vector Database

คำนวณ 6C Vector สำหรับแต่ละกลยุทธ์โดยการสร้าง embedding ของข้อความกลยุทธ์, คำนวณ cosine similarity กับ 6C definitions, Normalize เป็นสัดส่วนรวม 100% และบันทึกเป็น pickle file

5. การสร้างระบบ Semantic Search

พัฒนา 2 ระดับการค้นหา: Semantic Search (แปลง query เป็น embedding, คำนวณ cosine similarity) และ Reranking ด้วย 6C Framework (คำนวณ 6C pattern ของ query, เปรียบเทียบกับกลยุทธ์)

6. การคำนวณ Metrics แบบครอบคลุม

สร้างระบบประเมินผลที่วัดได้หลายมิติ: Semantic Similarity, 6C Similarity, Selection Score, Confidence Score, KL Divergence และ Match Ratio

4. สถาปัตยกรรมเว็บแอปพลิเคชัน

โครงสร้างระบบแบบ Multi-Tier Architecture

ระบบประกอบด้วย 3 ชั้นหลัก: Frontend (Vue.js), Backend Services (FastAPI + Express.js), และ AI/ML Layer

Frontend Layer

  • • Vue.js 3 - Modern JavaScript Framework
  • • Vite - Build Tool & Dev Server
  • • Vue Router 4 - Client-side Routing
  • • Fomantic UI - CSS Framework
  • • Axios - HTTP Client

Backend Services

  • • FastAPI - Primary Python API
  • • Express.js - Secondary Node.js API
  • • CORS - Cross-Origin Resource Sharing
  • • Uvicorn - ASGI Server
  • • Mongoose - MongoDB ODM

AI/ML Layer

  • • PyTorch - Deep Learning Framework
  • • Llama 3.2-1B - Language Model
  • • NumPy - Numerical Computing
  • • Gymnasium - RL Environment
  • • llama-cpp-python - Local Inference

Data Flow และ Component Communication

Client-Server Communication

→ Frontend (Port 5173) ↔ FastAPI (Port 8000)
→ REST API endpoints for game logic
→ Real-time AI strategy analysis
→ Express.js (Port 5000) for auxiliary services

AI Processing Pipeline

→ Game State → AI Model Inference
→ Strategy Analysis → LLM Processing
→ Vector Database → Semantic Search
→ Results → Frontend Display
